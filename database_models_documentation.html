外键关联关系总结

  • 用户相关: UserProfile → User, LoginLog → User, UserSession → User, Role权限关联
  • 组织架构: Department自关联(parent), Employee → User/Department/Position, Position → Department
  • 菜单管理: Menu自关联(parent), UserMenuConfig → User
  • IP管理: IPRecord → User(created_by), ScanTask → User(created_by)
  • 业务管理: BusinessSystem → User(owner/created_by), BusinessSystemDependency自关联
  • 资产管理: Asset → User(owner/custodian/created_by), Asset → Department, AssetChangeLog → Asset/User
  • 系统管理: SystemConfig → User(updated_by), AuditLog → User

数据库设计特点

  • 统一主键: 大部分表使用UUID作为主键，确保全局唯一性
  • 审计字段: 包含created_at、updated_at、created_by等审计字段
  • 软删除: 部分表支持软删除机制
  • JSON字段: 使用JSONField存储复杂配置和扩展信息
  • 状态管理: 通过status字段管理记录状态
  • 权限控制: 基于角色的权限管理体系
